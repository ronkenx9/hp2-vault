HP2-Vault is a next-generation, yield-bearing escrow protocol designed for the high-velocity agentic economy. It transforms locked merchant capital from a "dead" asset into a "living" yield-generating liquidity pool, utilizing an autonomous AI Jury to resolve disputes with cryptographic finality on the HashKey Settlement Protocol (HSP).

---

## 🚀 The PayFi Vision: Bridging the Efficiency Gap
Traditional Web3 escrow is a frictional bottleneck. When a merchant or machine agent locks funds during a trade, that capital becomes "dead"—it sits idle for days or weeks, losing value while waiting for delivery confirmation. In an era where AI agents transact in milliseconds, this idle capital represents a multi-billion dollar opportunity cost.

**HP2-Vault solves this by:**
1.  **Eliminating Opportunity Cost**: Escrowed funds aren't just safe; they're active. They accrue real-time yield through HashKey's local liquidity primitives, neutralizing the cost of security.
2.  **Autonomous AI Jury**: A 5-node consensus engine (Llama-3-70B/Groq) deliberates in a live terminal, scraping on-chain telemetry to issue 3/5 multi-sig verdicts in seconds.
3.  **Institutional Finality**: Fully integrated with the **HashKey Settlement Protocol (HSP)**, providing HSM-grade security and compliant cross-border settlement.

---

## 🛠️ Architecture: Institutional Security at Scale

### 1. The Resolution Layer (`/contracts`)
- **`HP2Vault.sol`**: The self-settling vault engine. Handles yield-bearing deposits and reads verdict state from the Arbitrator.
- **`HP2Arbitrator.sol`**: A pure state-machine oracle. Implements 3/5 threshold logic where "signatures" are generated by authenticated AI nodes.

### 2. The Verdict Engine (`/jury`)
- **Evidence Scraper**: Scans HashKey Chain logs and IPFS metadata for cryptographically verifiable delivery proof.
- **Consensus Orchestrator**: Manages the 5-node deliberation loop, ensuring 100% autonomous resolution without human intervention.
- **ECDSA Signer**: Nodes generate `secp256k1` signatures (`ecrecover` compatible) to trigger on-chain settlement.

### 3. Cyber-Merchant Console (`/merchant-portal`)
- **Institution-Grade UI**: Editorial Brutalist design built with Next.js 14 and Three.js.
- **Live Terminal**: A real-time command center window into the AI Jury's decision-making process.
- **Yield Pulse**: Persistent visual feedback indicating capital acceleration in the "Engine" zone.

## 🏁 Technical Implementation

### Local Development
```bash
# 1. Install dependencies
npm install

# 2. Run Hardhat Node
npx hardhat node

# 3. Start the Portal
cd merchant-portal
npm run dev
```

### AI Engine Setup
The Jury requires a `GROQ_API_KEY` for deliberation:
```bash
# In /jury/.env
GROQ_API_KEY=your_key_here
```
